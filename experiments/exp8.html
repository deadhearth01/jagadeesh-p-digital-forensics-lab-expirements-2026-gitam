EXPERIMENT 8

Usage of the Wireshark tool

Aim

To capture and analyze network traffic using Wireshark for forensic investigation and network troubleshooting purposes.

Theory

Wireshark is the world's most popular network protocol analyzer. It lets you capture and interactively browse the traffic running on a computer network. It is used for network troubleshooting, analysis, software and protocol development, and education.

In digital forensics, Wireshark is invaluable for analyzing network-based attacks, identifying data exfiltration, examining malware communication, and reconstructing network events. Captured traffic can reveal what data was transmitted, to which servers, and when.

Wireshark can dissect hundreds of protocols and display packet details in a human-readable format. It supports various capture filters and display filters to focus on specific traffic. The tool can also follow TCP streams to reconstruct complete conversations.

Tools Required

  • Wireshark (free download from wireshark.org)
  • Npcap or WinPcap (packet capture driver)
  • Active network connection
  • Administrator privileges

Procedure

  1. Download and install Wireshark from the official website
  2. During installation, also install Npcap for packet capture
  3. Launch Wireshark as Administrator
  4. Select the network interface to capture (Wi-Fi or Ethernet)
  5. Click the blue shark fin icon to start capture
  6. Generate some network traffic by browsing websites
  7. Click the red square to stop capture after collecting enough data
  8. Use display filters to narrow down traffic:
    • http - show only HTTP traffic
    • ip.addr == 192.168.1.1 - filter by IP address
    • tcp.port == 80 - filter by port
    • dns - show only DNS queries
  9. Right-click on a packet and select "Follow TCP Stream" to see full conversation
  10. Analyze packet details in the middle pane
  11. Export captured data or specific packets for documentation
  12. Save the capture file (.pcapng) for later analysis

Result

Network traffic was successfully captured and analyzed using Wireshark. The tool allowed examination of various protocols including HTTP, DNS, and TCP. Using display filters and the Follow Stream feature, specific network conversations were reconstructed. This demonstrates how Wireshark can be used for network forensics to investigate suspicious network activity.
