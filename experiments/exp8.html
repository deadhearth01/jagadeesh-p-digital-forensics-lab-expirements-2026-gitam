Experiment 08

Usage of the Wireshark Tool for Network Packet Analysis

🎯 Aim

To capture and analyze live network packets using Wireshark, apply display filters to isolate specific traffic, and identify forensic evidence within network communication data.

📖 Theory

Wireshark is the world's most widely used network protocol analyzer. It lets you capture and interactively browse the traffic running on a computer network. It is used by network engineers, security professionals, and forensic investigators to diagnose network problems and detect malicious activity.

Wireshark captures packets at the Data Link Layer (Layer 2) using the system's network interface in promiscuous mode, allowing it to capture all packets on the network segment, not just those addressed to the local machine.

Key concepts in Wireshark forensics:

  • PCAP files — Packet capture format; the standard for saving captured traffic
  • Display Filters — Powerful expressions to filter displayed packets (e.g., http, ip.addr == 10.0.0.1)
  • Capture Filters — BPF (Berkeley Packet Filter) syntax applied during capture to limit what is recorded
  • Follow TCP/HTTP Stream — Reconstructs the full conversation between two hosts
  • Statistics → Protocol Hierarchy — Shows a breakdown of all protocols in the capture
  • Expert Information — Flags anomalies, errors, and warnings automatically

From a forensic standpoint, Wireshark is used to detect port scanning, ARP spoofing, DNS poisoning, data exfiltration, C2 (command-and-control) communications, and unencrypted credential transmission.

🛠️ Tools Required

Wireshark (v4.x) — The primary packet capture and analysis tool.

Npcap — Windows packet capture driver required by Wireshark.

A network interface — Wi-Fi or Ethernet adapter to capture traffic from.

📋 Procedure

  1. Launch Wireshark and Select Interface

    Open Wireshark. The welcome screen shows all available network interfaces with a traffic activity graph. Double-click the active interface (e.g., Wi-Fi or Ethernet) to begin capture.

  2. Generate Test Traffic

    Open a browser and visit an HTTP website (not HTTPS) to generate visible traffic. Alternatively, use ping google.com in Command Prompt to generate ICMP packets.

  3. Stop the Capture

    Click the red Stop button after 30 seconds. The captured packets are now displayed in the main panel. Save the capture as lab_capture.pcap via File → Save As.

  4. Apply Display Filters

    Use the filter bar to isolate specific protocols:

    # Show only HTTP traffic http # Show traffic to/from a specific IP ip.addr == 142.250.190.78 # Show only DNS queries dns # Show TCP handshakes (SYN packets) tcp.flags.syn == 1 and tcp.flags.ack == 0

  5. Follow an HTTP Stream

    Right-click any HTTP packet and select Follow → HTTP Stream. This reconstructs the full request-response conversation, showing HTTP headers, request body, and server response in plain text.

  6. Analyse Protocol Hierarchy

    Go to Statistics → Protocol Hierarchy. This shows the percentage of traffic belonging to each protocol, helpful for identifying unusual protocol usage.

  7. Examine IO Graph

    Go to Statistics → IO Graph. This visualises packet rate over time. Traffic spikes may indicate a data transfer, port scan, or DDoS event.

📝 Observations & Result

Wireshark successfully captured live network traffic. HTTP stream reconstruction revealed request headers including the Host, User-Agent, and response status codes. DNS queries showed domain resolution for visited websites. The Protocol Hierarchy revealed that the majority of traffic was TCP, with HTTP and DNS as the top application-layer protocols. The IO Graph showed a clear spike during the browser session.

Result: Network packets were successfully captured and analyzed using Wireshark, demonstrating its capability as a forensic tool for network traffic inspection and evidence collection.
