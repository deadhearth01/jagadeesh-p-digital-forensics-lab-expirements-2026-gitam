Experiment 06

Extraction of Browser Artefacts

🎯 Aim

To extract and analyze browser artifacts — including browsing history, cookies, cache files, saved passwords, and download records — from major web browsers for forensic investigation.

📖 Theory

Web browsers store a significant amount of user activity data locally on the system. These browser artifacts are valuable forensic evidence that can reveal websites visited, search queries, downloaded files, form data, and even login credentials.

Key browser artifact types and their storage locations on Windows:

  • History — URLs and visit timestamps stored in SQLite databases (History file in Chrome)
  • Cookies — Session tokens and tracking cookies (Cookies SQLite DB)
  • Cache — Locally stored web page content for faster loading
  • Downloads — Record of all files downloaded (History DB, downloads table)
  • Saved Passwords — Encrypted credentials stored in Login Data (Chrome) or logins.json (Firefox)
  • Bookmarks — User-saved URLs indicating sites of interest
  • Search Terms — Queries entered into the browser's address bar (omnibox)

Chrome and Edge store artifacts in SQLite databases under %LOCALAPPDATA%\Google\Chrome\User Data\Default\. Firefox stores data in %APPDATA%\Mozilla\Firefox\Profiles\[profile]\. These files can be parsed even when a user has cleared their browser history, as forensic tools can recover deleted SQLite records.

SQLite Forensics Chrome / Firefox Browsing History Cookies BrowsingHistoryView

🛠️ Tools Required

BrowsingHistoryView (NirSoft) — Aggregates browsing history from Chrome, Firefox, Edge, and IE into a single view.

DB Browser for SQLite — Opens and queries SQLite database files directly.

Hindsight — Chrome / Chromium artifact parser that reconstructs browsing sessions.

Autopsy — Can automatically parse browser artifacts from a forensic image.

📋 Procedure

  1. Locate Browser Profile Directories

    Open File Explorer and navigate to the Chrome profile directory:
    C:\Users\[User]\AppData\Local\Google\Chrome\User Data\Default\
    Note the files: History, Cookies, Login Data, Bookmarks.

  2. Copy Artifact Files

    Copy the SQLite database files to a separate working directory. Do not open them while Chrome is running as the browser locks the files.

  3. Open History DB in SQLite Browser

    Open History in DB Browser for SQLite. Run the following query to view recent visits:

    SELECT url, title, visit_count, datetime(last_visit_time/1000000-11644473600, 'unixepoch') AS last_visit FROM urls ORDER BY last_visit_time DESC LIMIT 50;

  4. Run BrowsingHistoryView

    Launch BrowsingHistoryView, configure it to scan all browsers, and click OK. The tool aggregates history from all installed browsers into a sortable table. Export as CSV.

  5. Extract Download Records

    In DB Browser, switch to the downloads table in the History DB. Note file names, target paths, start times, and referrer URLs.

  6. Analyse Cookies

    Open Cookies in DB Browser. Query the cookies table to identify session tokens, third-party trackers, and logged-in services.

  7. Run Hindsight for Full Session Reconstruction

    Run Hindsight against the Chrome profile directory to produce a comprehensive HTML report including page transitions and search terms.

📝 Observations & Result

Browser artifacts were successfully extracted from Chrome's SQLite databases. The history database revealed all recently visited URLs with timestamps. Download records showed file names, download paths, and source URLs. BrowsingHistoryView consolidated history from multiple browsers. Hindsight produced a detailed timeline reconstruction of the user's browsing session.

Result: Browser artifacts were successfully extracted and analyzed, providing a comprehensive view of the user's online activity — demonstrating the forensic value of browser artifact analysis.
