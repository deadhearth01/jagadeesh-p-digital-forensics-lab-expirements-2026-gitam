Experiment 09

Using the Sysinternals Suite for Network Tracking and Process Monitoring

🎯 Aim

To use the Microsoft Sysinternals suite — specifically Process Monitor (Procmon), Process Explorer, and TCPView — to monitor running processes, track system activity, and observe network connections in real time for forensic investigation.

📖 Theory

The Sysinternals Suite is a collection of advanced Windows system utilities developed by Mark Russinovich and Bryce Cogswell, now maintained by Microsoft. It is an indispensable toolkit for Windows forensic investigators, incident responders, and malware analysts.

Key tools in the Sysinternals suite:

  • Process Monitor (Procmon) — Monitors file system, registry, network, and process/thread activity in real time with extensive filtering capabilities
  • Process Explorer — An advanced task manager that shows detailed information about running processes, including DLLs loaded, parent-child relationships, and VirusTotal integration
  • TCPView — Displays all active TCP and UDP endpoints with the associated process name and PID
  • Autoruns — Shows all programs configured to run at startup — critical for detecting malware persistence
  • PsTools — Command-line tools for managing processes and services on local or remote machines

In a forensic context, Sysinternals tools are used during live response investigations to capture volatile data — running processes, active network connections, registry changes, and file system activity — that would be lost after a system reboot.

Process Monitor's ability to record every file, registry, and network event (with timestamps, call stacks, and process details) makes it uniquely powerful for tracking malware behavior and identifying indicators of compromise (IOCs).

🛠️ Tools Required

Sysinternals Suite — Downloadable from Microsoft (live.sysinternals.com or as a ZIP). Key utilities: procmon.exe, procexp.exe, tcpview.exe, autoruns.exe.

Windows OS (7/10/11) — The environment in which the tools operate.

Administrator privileges — Required for kernel-level process monitoring.

📋 Procedure — Process Monitor

  1. Launch Process Monitor as Administrator

    Right-click Procmon.exe and choose "Run as Administrator". Accept the EULA. Procmon immediately starts capturing all system events.

  2. Apply a Filter to Focus on a Process

    Go to Filter → Filter (Ctrl+L). Add a filter: Process Name → is → chrome.exe → Include. Click Add, then OK. Now only Chrome's activity is shown.

  3. Observe File and Registry Activity

    Open Chrome and navigate to a website. Observe Procmon capturing file reads/writes, registry accesses, and network connections generated by Chrome. Note the file paths and registry keys accessed.

  4. Save the Procmon Log

    Stop capture (Ctrl+E) and save the log as a .PML file for later analysis. This log can be loaded offline for investigation.

📋 Procedure — TCPView

  1. Launch TCPView as Administrator

    Run tcpview.exe as Administrator. The tool immediately lists all active TCP/UDP connections with process name, PID, local address, remote address, and state.

  2. Identify Active Connections

    Look for ESTABLISHED TCP connections. Note which processes are communicating with external IP addresses. Connections to unknown remote IPs may indicate malware C2 communications.

  3. Kill a Suspicious Connection

    Right-click any suspicious connection and select Close Connection. This terminates the TCP session without ending the process.

  4. Document All Active Connections

    Go to File → Save to export all active connections as a .txt file for the forensic report.

📝 Observations & Result

Process Monitor recorded all file system, registry, and network events generated by the monitored process. TCPView revealed all active network connections along with the owning process, enabling identification of any unauthorized outbound connections. Process Explorer provided a detailed process tree showing parent-child relationships and loaded DLLs, which is essential for detecting process injection and malware masquerading as legitimate processes.

Result: The Sysinternals suite was used effectively for live system forensics — monitoring process activity, network connections, and system events in real time, demonstrating its critical role in incident response and forensic investigation.
