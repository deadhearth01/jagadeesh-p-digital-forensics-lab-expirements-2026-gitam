EXPERIMENT 5

Collection of the Email evidence in the victim's PC

Aim

To collect and preserve email evidence from a computer system for forensic analysis and investigation.

Theory

Email forensics involves the examination of email messages and their metadata to gather evidence for legal proceedings. Emails contain valuable information including sender and recipient addresses, timestamps, IP addresses, message content, and attachments.

Email clients like Microsoft Outlook store messages in specific file formats. Outlook uses PST (Personal Storage Table) files to store emails, contacts, and calendar items locally. These PST files can be analyzed to recover deleted emails and examine communication patterns.

Email headers contain technical information about how the message was routed across servers. This includes the originating IP address, mail servers involved, timestamps at each hop, and authentication results. Analyzing headers can help identify spoofed emails or trace the true sender.

Tools Required

  • Microsoft Outlook (for accessing PST files)
  • Kernel Outlook PST Viewer (free PST viewer)
  • Email header analyzer (online tools available)
  • FTK Imager (for creating forensic copy of email files)

Procedure

  1. Locate the email data files on the system. For Outlook, check: C:\Users\[Username]\Documents\Outlook Files\
  2. Create a forensic copy of the PST/OST files before analysis
  3. Open the PST file using Kernel Outlook PST Viewer or similar tool
  4. Browse through the folder structure: Inbox, Sent Items, Deleted Items, etc.
  5. Identify relevant emails based on date range, sender, or keywords
  6. For each important email, view the full header information
  7. Copy the email header and analyze using an online header analyzer
  8. Document findings including sender IP, routing information, and timestamps
  9. Export relevant emails in a standard format (EML or MSG) for evidence
  10. Create a report summarizing the email evidence found
