Experiment 04

Creating a Forensic Image of a Hard Drive Using FTK Imager

🎯 Aim

To create a forensically sound, bit-for-bit image of a hard drive using FTK Imager and to verify the integrity of the image using MD5 and SHA-1 hash values.

📖 Theory

In digital forensics, investigators must never work directly on original evidence. Instead, a forensic image — an exact, sector-by-sector duplicate of the original storage media — is created and all analysis is performed on the copy. This preserves the integrity of the original evidence and ensures admissibility in court.

FTK Imager (Forensic Toolkit Imager) by AccessData is one of the most widely used tools for creating forensic images. It supports multiple image formats including:

  • E01 (Expert Witness Format) — Compressed, with embedded case metadata and hash verification
  • RAW / DD — Uncompressed bit-for-bit copy
  • AFF (Advanced Forensics Format) — Open standard with metadata support

Hash Verification is fundamental to forensic imaging. Before imaging, the hash (MD5/SHA-256) of the source drive is computed. After imaging, the hash of the image is computed and compared. Matching hashes prove that the copy is identical to the original — a requirement for evidence admissibility under the best evidence rule.

A write blocker (hardware or software) must be used to prevent any writes to the source drive during imaging, which would alter evidence and invalidate the hash.

FTK Imager E01 Format MD5 / SHA-1 Write Blocker Chain of Custody

🛠️ Tools Required

FTK Imager (v4.x) — Free forensic imaging utility from AccessData / Exterro.

Hardware Write Blocker — Or enable FTK Imager's software write-blocking feature to protect source media.

Source Drive — USB drive or secondary HDD to be imaged.

Destination Drive — Separate storage with sufficient space to hold the image.

📋 Procedure

  1. Connect the Source Drive via Write Blocker

    Attach the source USB or HDD through a hardware write blocker, or use FTK Imager's built-in software write protection. This ensures no data is written to the source during acquisition.

  2. Launch FTK Imager

    Open FTK Imager as Administrator. Go to File → Create Disk Image.

  3. Select Source Evidence Type

    Choose Physical Drive to image the entire drive at the physical sector level. Select the appropriate drive from the list (e.g., \\PHYSICALDRIVE1).

  4. Configure Image Destination

    Click Add to set the output. Choose E01 as the image type. Enter case details (Case Number, Evidence Number, Examiner, Notes) and set the destination folder on the evidence drive.

  5. Enable Hash Verification

    Ensure Verify images after they are created and Create directory listings are checked. Select both MD5 and SHA-1 hash algorithms.

  6. Start Imaging

    Click Start. Monitor the imaging progress. FTK Imager displays the elapsed time, sectors read, and estimated completion time.

  7. Verify Hash Values

    Upon completion, FTK Imager computes and displays the source and image hash values. Confirm they match:

    Source MD5: d41d8cd98f00b204e9800998ecf8427e Image MD5: d41d8cd98f00b204e9800998ecf8427e ✓ Hash values match — Image verified.

  8. Save the Image Report

    FTK Imager generates a .txt summary report. Save it alongside the E01 image as part of the chain of custody documentation.

📝 Observations & Result

The forensic image was created successfully in E01 format. The MD5 and SHA-1 hash values of the source drive and the resulting image were identical, confirming the integrity of the acquisition. The image file was accompanied by a verified case report suitable for chain-of-custody documentation.

Result: A forensically sound disk image was successfully created using FTK Imager, with hash verification confirming that no data was modified during the acquisition process.
