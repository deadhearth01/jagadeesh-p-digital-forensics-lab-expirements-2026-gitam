EXPERIMENT 4

Creating the forensic image of the hard drive using FTK Imager forensics

Aim

To create a bit-by-bit forensic image of a storage device using FTK Imager and verify its integrity using hash values.

Theory

Forensic disk imaging is the process of creating an exact bit-for-bit copy of a storage device. Unlike a simple file copy, a forensic image captures everything on the drive including deleted files, file slack, and unallocated space. This ensures no evidence is missed during investigation.

FTK Imager is a free forensic imaging tool developed by AccessData. It can create forensic images in various formats including E01 (EnCase format), DD (raw), and AFF. The tool also generates hash values (MD5 and SHA1) to verify that the image is an exact copy of the original.

Hash verification is crucial in forensics because it proves the evidence has not been tampered with. If the hash of the image matches the hash of the original drive, we can confirm the image is authentic.

Tools Required

  • FTK Imager (free download from AccessData)
  • Source drive or USB device to image
  • Destination drive with sufficient storage space
  • Write blocker (recommended for actual forensic work)

Procedure

  1. Download and install FTK Imager from the official website
  2. Connect the source drive (USB or external HDD) to the computer
  3. Launch FTK Imager as Administrator
  4. Click File > Create Disk Image
  5. Select the source type (Physical Drive, Logical Drive, or Image File)
  6. Choose the source drive from the list
  7. Select destination and image format (E01 recommended)
  8. Enter case information: Case Number, Evidence Number, Examiner Name, Description
  9. Choose compression level and fragment size if needed
  10. Click Start to begin the imaging process
  11. Wait for the process to complete - this may take time depending on drive size
  12. Verify the hash values match between source and image

Result

A forensic image of the storage device was successfully created using FTK Imager. The MD5 and SHA1 hash values of the created image matched the hash values of the original source drive, confirming that the image is an exact bit-by-bit copy. This forensic image can now be used for investigation without risking damage to the original evidence.
