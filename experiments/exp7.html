Experiment 07

Live Forensic Case Investigation Using Autopsy

🎯 Aim

To conduct a complete forensic case investigation using the Autopsy digital forensics platform — creating a case, ingesting a disk image, running ingest modules, and producing a final forensic report.

📖 Theory

Autopsy is a free, open-source digital forensics platform developed by Basis Technology. It is widely used by law enforcement agencies, military organizations, and corporate investigators. Autopsy provides a graphical interface over The Sleuth Kit (TSK) — a collection of open-source forensic tools.

Autopsy's ingest modules automatically analyze a disk image or live device and extract:

  • File System Analysis — Parses NTFS, FAT, EXT2/3/4, HFS+ file systems
  • Keyword Search — Searches the entire disk for specific strings or regex patterns
  • Hash Lookup — Identifies known malicious files using NSRL or custom hash sets
  • Web Artifacts — Browser history, cookies, and bookmarks
  • Email Artifacts — PST/MBOX parsing
  • EXIF Parser — GPS and device metadata from images
  • Recent Activity — Windows event logs, registry hives, prefetch files
  • Android Analyzer — Mobile device artifacts from Android images

The Autopsy interface uses a case-based workflow consistent with legal evidence management. Each case has a unique identifier, and all actions are logged in the case database for reproducibility and admissibility.

🛠️ Tools Required

Autopsy (v4.x) — The primary investigation platform.

A Disk Image (E01 / DD) — The evidence file created in Experiment 04, or a sample forensic image.

Java Runtime Environment (JRE) — Required by Autopsy.

📋 Procedure

  1. Create a New Case

    Open Autopsy and click New Case. Enter the case name (e.g., DF_Lab_Case01), base directory, case number, and examiner name. Click Finish.

  2. Add a Data Source

    Choose Disk Image or VM File as the data source type. Browse to the E01 or DD image created in Experiment 04. Click Next.

  3. Configure Ingest Modules

    Select the ingest modules to run: Recent Activity, Hash Lookup, File Type Identification, Keyword Search, Email Parser, EXIF Parser, and Extension Mismatch Detector. Click Finish to begin ingestion.

  4. Monitor Ingestion Progress

    The bottom progress bar shows ingestion status. Wait for all modules to complete. This may take several minutes depending on image size.

  5. Explore the Results Tree

    In the left panel, explore the tree: Data Artifacts → Web History, Deleted Files, Email Messages, EXIF Metadata, Interesting Items, etc.

  6. Use the Timeline Viewer

    Go to Tools → Timeline. Switch to Counts View to see file activity over time. Identify spikes in file creation, modification, or access that correspond to suspicious events.

  7. Tag Key Evidence

    Right-click on items of interest and select Add File Tag → Notable Item. Add descriptive comments to support the forensic narrative.

  8. Generate the Final Report

    Go to Generate Report. Choose HTML Report. The report will include all tagged items, ingest results, and case metadata. Save it as the final forensic case report.

📝 Observations & Result

Autopsy successfully ingested the disk image and all configured modules ran without errors. The results tree populated with web history, deleted files, email artifacts, EXIF data, and keyword search hits. The timeline view revealed a clear pattern of file access. Tagged evidence items were included in the HTML forensic report, which documented the complete investigation workflow.

Result: A complete forensic case investigation was conducted using Autopsy, demonstrating the platform's ability to automate artifact extraction and generate a court-admissible forensic report.
