Experiment 01

Recovering Deleted Files Using Forensic Tools

🎯 Aim

To recover deleted files from a storage device using digital forensic tools such as Recuva and Autopsy, and to understand how deleted data can be retrieved for forensic investigation purposes.

📖 Theory

When a file is deleted from a storage device, the operating system does not immediately erase the actual data. Instead, it marks the space occupied by the file as available for reuse and removes the file's entry from the file allocation table (FAT) or Master File Table (MFT). The actual data remains on the disk until it is overwritten by new data.

Digital forensic investigators exploit this behavior to recover deleted files. Tools like Recuva and Autopsy scan the raw disk sectors, reconstruct the file system structures, and locate file signatures (magic bytes) to carve out recoverable files.

File Carving is a technique used to extract files from raw data by searching for known file headers and footers without relying on file system metadata. For example, JPEG images begin with the hex signature FF D8 FF and end with FF D9.

Forensic recovery is a critical step in cybercrime investigations. Evidence such as deleted documents, images, emails, and database records can be recovered from hard drives, USB drives, memory cards, and solid-state drives (SSDs). However, SSDs use TRIM commands that may make recovery harder compared to traditional HDDs.

🛠️ Tools Required

Recuva — A free Windows utility by Piriform that scans storage media for deleted files and recovers them with a user-friendly interface.

Autopsy — An open-source digital forensics platform used by law enforcement and investigators to analyze disk images and recover deleted data.

Test Storage Device — A USB drive or hard disk partition with intentionally deleted files for demonstration.

📋 Procedure

  1. Prepare the Test Environment

    Copy sample files (images, documents, text files) onto a USB drive or test partition. Delete several files using the standard OS delete operation and also empty the Recycle Bin.

  2. Launch Recuva

    Open Recuva and select the drive letter corresponding to your test device. Choose "All Files" in the file type wizard and click "Start" to begin the scan.

  3. Review Scan Results

    Recuva displays a list of recoverable files colour-coded by recoverability: green (excellent), orange (poor), red (unrecoverable). Note which files are still recoverable and their original paths.

  4. Recover Selected Files

    Select the files you wish to recover, click "Recover", and choose a destination folder different from the source drive to avoid overwriting evidence.

  5. Create a Disk Image with Autopsy

    Open Autopsy and create a new case. Add the USB drive as a data source. Autopsy will ingest the drive and index all recoverable files including deleted items.

  6. Analyse Results in Autopsy

    Navigate to the "Deleted Files" section in the tree view. Examine file metadata such as timestamps, file size, and parent directory. Export relevant files as evidence.

  7. Document Findings

    Record the number of files recovered, their types, and recoverability status. Generate the Autopsy case report in HTML or PDF format.

📝 Observations & Result

Files that were recently deleted and had not been overwritten were successfully recovered using both Recuva and Autopsy. Files deleted earlier and partially overwritten showed degraded recovery quality. Autopsy provided richer metadata including original file paths and MAC (Modified, Accessed, Created) timestamps, which are critical for forensic timelines.

Result: Deleted files were successfully recovered using Recuva and Autopsy, demonstrating that standard file deletion does not permanently erase data from storage media.
